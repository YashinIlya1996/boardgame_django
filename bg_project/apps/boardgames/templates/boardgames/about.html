О проекте:

Данный проект реализован в рамках изучения python в целом и фрeймворка Django в частности.

Используемая БД – PostgreSQL.

Стандартная модель пользователя django.contrib.auth.models.User расширена путем One-to-one связи с моделями Profile и Wishlist.

Для завершения регистрации пользователю необходимо подтвердить введенный на этапе регистрации email – на почту направляется проверочный код, который необходимо ввести на завершающем этапе регистрации. Зарегистрированный и аутентифицированный пользователь получает возможность редактировать состав своего Wishlist’а, создавать новые встречи и подавать запросы на участие в уже созданных встречах, отправлять запросы на добавление пользователей во френдлист.

Добавление других пользователей во френдлист дает возможность отслеживать их активность на сайте: посредством сигналов запускаются callback-функции на создание уведомлений пользователям, находящимся в списке друзей (используются post_save, post_delete, m2m_changed сигналы), например, при создании другом встречи или при добавлении им новой игры в Wishlist. Подобным образом пользователю направляются уведомления об изменении его статуса во встречах (принятие, отклонение запроса, удаление из состава игроков) и статуса дружбы с другими пользователями.

Выполнение периодических и занимающих долгое время задач реализовано с помощью Celery в связке с Redis в качестве брокера/бэкэнда. На данный момент Celery выполняет следующие задачи:

  • Раз в сутки парсинг тематических сайтов и добавление в БД новых настольных игр;
  • Отправка пользователю письма для подтверждения email при регистрации;
  • Отправка писем пользователям, участвующим во встрече, за определенное время до начала встречи.

Сведения обо всех играх доступны также через API (/api/v0/games/), реализация с Django REST Framework.

Фронтенд реализован без использования CSS-фреймфорков (Bootstrap и пр.)

Деплой сайта – свзяка Nginx + Gunicorn. Автоматический запуск gunicorn и Celery (workers, beat) обеспечивает настроенный менеджер процессов supervisor. Присутствует SSL.

Дальнейшие планы по усовершенствованию проекта:

  • Добавление нового функционала (комментарии к играм и встречам, фильтры поиска, настройка отображения данных другим пользователям)
  • Изучение JavaScript и на более глубоком уровне DRF (для реализации отдельного функционала через AJAX-запросы);
  • Рефакторинг кода и оптимизация запросов к БД;
  • Приведение интерфейса в более приятный вид и адаптация под мобильные экраны.
